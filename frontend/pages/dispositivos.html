Guia de Integração — Leitor IDUHF Control ID

1. Configuração de Rede do Equipamento

  1. Conecte o leitor IDUHF à mesma rede local do servidor (switch ou roteador).
  2. Acesse a interface web do equipamento pelo IP padrão de fábrica: 192.168.0.1 (usuário: admin, senha: admin).
  3. Vá em Configurações → Rede e defina um IP fixo na sua sub-rede (ex: 192.168.1.100).
  4. Certifique-se de que o servidor do ERP consegue alcançar esse IP (mesma rede ou roteamento configurado).

2. Modo de Operação da TAG UHF

O IDUHF suporta dois modos para TAGs UHF:

ModoObjeto na APIFormato da TAGQuando usar
Extended (recomendado) uhf_tags String hexadecimal (ex: CAFEDAD0) TAGs com EPC de 96 bits — padrão do IDUHF
Standard cards Número inteiro (Wiegand) Compatibilidade com leitores RFID tradicionais

O sistema tenta primeiro o modo Extended. Se falhar, tenta o modo Standard automaticamente.

3. Formato das TAGs no ERP

No cadastro de veículos, o campo TAG deve conter:

  • Modo Extended (IDUHF): o código EPC da tag em hexadecimal, ex: MAJJHAG0022 ou E2003411B802011820A0B8F3
  • Modo Standard: o número Wiegand da TAG, ex: 12345
Como obter o código EPC da TAG: Aproxime a TAG do leitor IDUHF e consulte os logs de acesso no equipamento (Controle de Acesso → Logs) — o campo uhf_tag mostrará o código EPC lido.

4. Fluxo de Integração

1
Cadastre o dispositivo na aba "Dispositivos" com IP, porta e credenciais.
2
Teste a conexão — o sistema faz login na API do equipamento e confirma a comunicação.
3
Sincronize as TAGs — o sistema envia todos os veículos cadastrados como usuários no equipamento, vinculando cada TAG UHF.
4
Coleta automática de logs — ao clicar em "Coletar Logs", o sistema importa os acessos registrados pelo equipamento e os registra no controle de acesso do ERP.
5
Novos veículos — ao cadastrar um novo veículo com TAG, clique em "Sincronizar TAGs" novamente. O sistema enviará apenas os veículos ainda não sincronizados.

5. Requisitos e Observações

  • O servidor do ERP (HostGator) deve ter acesso de rede ao IP do equipamento. Em redes locais, pode ser necessário configurar um proxy reverso ou usar o módulo de integração local.
  • A API do Control ID usa HTTP (porta 80 por padrão) — certifique-se de que firewalls não bloqueiam essa porta.
  • O token de sessão da API expira em ~30 minutos. O sistema faz login automaticamente a cada operação.
  • O grupo de acesso padrão usado é o Grupo 1 do equipamento. Certifique-se de que ele tem regras de acesso configuradas no equipamento.
